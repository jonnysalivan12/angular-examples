Agentic Workflow — End-to-End Flow

8 etapów · agent per krok · human-in-the-loop gates

📥 USER STORY (od analityka)
▼
Etap 1
Doprecyzowanie
Clarify
▸ Agent: Clarifier (orchestrator)
  • domain validator — kanoniczne pojęcia
  • 6 fal pytań × max 8 + 2 fale konfliktów
  • AC Writer Translator → Writer Redaktor
  • Requirements Writer
out: input-story.md · qa.md · coverage-metrics.md · requirements.json ✋ gate: Analityk
▼
Etap 2
Klasyfikacja
Classify
▸ Agent: Story Classifier (Sonnet)
  • typ: story / epic / spike / bug
  • epic → dekompozycja na sub-stories
  • routing do właściwej ścieżki workflow
out: classification.yaml
▼
Etap 3
Architektura
Architect
▸ Agent: Architect
  • mapa zmian — które pliki, dlaczego
  • API contracts · wpływ na NGXS store
  • ryzyka, blokery, pytania otwarte
out: architecture.md + change-map.json ✋ gate: Tech Lead
▼
Etap 4
Planowanie
Plan
▸ Agent: Planner
  • atomowe taski (≤ 2h) z mapy zmian
  • graf zależności: Models → Actions → State → Component
  • wzorzec referencyjny dla każdego pliku
out: tasks.json (ordered) ✋ gate: Dev
▼
Etap 5
Implementacja
Code
▸ Agent: Coder (per task, równolegle)
  • czyta skille — NGXS, NestJS, Nx
  • generuje kod wg wzorca z plannera
  • commit per task
out: diff / branch feature/*
▼
Etap 6
QA
Test & CI Guard
▸ Agent: Tester + CI Guard
  • unit (Jest/Jasmine) · e2e (Cypress)
  • tsc --noEmit · ESLint · build prod
  • max 3 auto-iteracje → eskalacja
↺ fail → back to Etap 5 z kontekstem błędu
▼
Etap 7
Review
Code & Security Review
▸ Agent: Reviewer + Security Scan
  • naming, lint, design system
  • XSS, hardcoded secrets, role guards
  • OWASP Top10 · CSP/SRI · dane w localStorage
↺ reject → back to Etap 5 z komentarzem ✋ gate: Dev + Security
▼
Etap 8
Dokumentacja
Docs
▸ Agent: Doc Writer
  • PR description — co / dlaczego / jak testować
  • changelog · ADR (jeśli zmiana architekturalna)
  • Storybook stories
out: PR ready ✋ gate: Dev
▼
✓ PR: Kod · Testy · Security · Docs · CI green  →  merge to develop

Zasady niezmienne

  • każdy agent ma kontekst: requirements.json + skille projektu
  • gate = explicit approve, nie timeout
  • max 3 auto-iteracje na błąd → eskalacja
  • audit trail każdej decyzji agenta
